    データパックのつくり方

    フォルダ構造¶

    データパックのフォルダの構造は次のようになっています。

    データパックフォルダ構造
    Datapacks/
├── data/
│   └── <namespace>/
│       ├── advancements/
│       │   └── advancements.json
│       ├── functions/
│       │   └── main.mcfunction
│       ├── loot_tables/
│       │   └── loot.json
│       ├── structures/
│       │   └── structure.nbt
│       ├── recipes/
│       │   └── recipe.json
│       └── tags/
│           ├── blocks
│           ├── items
│           ├── fluids
│           └── functions
└── pack.mcmeta
    データパックを構築する場合は、dataフォルダとpack.mcmetaは必須です。advancements、functions、loot_tables、structures、recipesは必要に応じて追加します。例えば、functionの機能のみを使用する場合は、それ以外のフォルダを作る必要はありません。

    名前空間(namespace)¶

    名前空間(namespace)は、データパックで既存のデータパックの上書きをしたり、追加するのに行われます。例えば、名前空間をminecraftにした場合、バニラのデータパックに上書きする形で作成するデータパックが動きます。minecraft以外にした場合、バニラのデータパックに追加される形で動きます。

    pack.mcmetaの記述¶

    次にpack.mcmetaの書き方について説明します。pack.mcmetaはファイルの中身はほぼリソースパックに使われるpack.mcmetaと同じです。 次はサンプルです。

    pack.mcmeta
    {
  "pack": {
    "pack_format": 7, # (1)
    "description": "データパックの説明"
  }
}
    1. この項目について注意点があります。詳しくはpack_formatの項目をご覧ください。

    pack_format¶

    pack_formatは、データパックを実装するMinecraftのワールドのバージョンによって値が変わります。また、1.17以降リソースパックのpack.mcmeta とデータパックのpack.mcmetaのpack_formatが分割されたため必ず一致するとは限りません。
    pack_formatの値がワールドのバージョンと異なる場合は、互換性に関する警告が表示されます。(ただしデータパックは問題なく実行できます)
    pack_formatに書き込む値については次の表をご覧ください。

    バージョン pack_formatの値
    1.19.4 12
    1.19~1.19.3 10
    1.18.2 9
    1.18~1.18.1 8
    1.17~1.17.1 7
    1.16.2~1.16.5 6
    1.15~1.16.1 5
    1.13~1.14.4 4

    description¶

    この項目にはデータパックの説明を入力します。/datapack list でリスト表示されたデータパックに説明が追加されます。

    開発環境の追加¶

    データパックを書く上で開発環境が必要になります。最低限メモ帳とエクスプローラーがあればデータパックは作成できますが、データパックにエラーがあった場合、シングルワールドではエラーは表示されません。そのため、どこでエラーが発生しているかがわからなくなるため、開発環境を作成するのは必要です。 次の項目より「Microsoft Visual Studio Code」を用いた開発環境の作成について紹介します。

    Microsoft Visual Studio Code(以後VSCode)の追加¶

    Microsoft Visual Studio Codeの配布ページより画像の赤枠にあるボタンからインストーラをダウンロードします。 次に自分が開発する環境に合わせてOSを選んでください。(Windows版の場合は一番左のWindows、Linux版の場合は使っているディストリビューションに合わせたパッケージファイルを、Mac版の場合は一番右のMacを選択) インストーラが起動しますので画面の指示に従ってインストールしてください。(基本的には「次へ」と「インストール」のみで完了します)

    wingetを用いたセットアップ

    wingetを用いてVSCodeをセットアップすることもできます。ただし、OSがWindows 10 1809以降である必要があります。 コマンドプロンプトを開き、 

    winget install vscode --source winget
    を入れて実行することで最新版のVSCodeをインストールすることができます。

    VSCodeの日本語化¶

    日本語化については、こちらのサイトが参考になります。 https://www.javadrive.jp/vscode/install/index4.html

    拡張機能の追加¶

    VSCodeでデータパックの色分け、便利機能の追加を行うには次の拡張機能が必要です。

    • Data-pack helper Plus
    • MC Datapack Utility

    これらの拡張機能を入れることによってmcfunctionが自動で色分けされるようになり、エラーも自動検出されます。また、データパックのテンプレート(フォルダ)を自動で作成する機能もあります。

    データパックの作成¶

    Visual Studio Codeを開いてCtrl+Shift+Pを押すと、画像のようなパネルが表示されます。 ここにDatapackと入れると、「Datapack:データパックテンプレートを作成」が検索に引っ掛かります。それをマウスでクリックするか選択してEnterを押します。 次に、次の画像のように画面が切り替わります。新規作成する場合は、「Create a datapack from a templete」を選択します。 そうすると、どこにそのデータパックを作成するかフォルダを選択する画面が出てくるのでデータパックを実装するワールドのdatapacksフォルダを指定してやるといいでしょう。
    その後、データパックの名前と説明、名前空間について聞かれます。datapack nameにはデータパックの名前を、datapack descriptionは、pack.mcmetaの記述にあるdescriptionの項目をご覧ください。名前空間には、本ページのnamespaceの項目をご覧ください。
    これらの操作をゆっくり確認したい場合、開発元のリポジトリのreadmeにありますのでご確認ください。
    MC Datapack Studio(Readme_ja.md)

    名前空間まで選択後、次の画面が表示されます。

    functionのみ使用する場合は、デフォルトでチェックがつけられている#load.json & ~~~:load.mcfunction、#tick.json & ~~~:tick.mcfunctionのみで問題ありません。(~~~は名前空間が入ります) recipes、advancements、loot_table、predicateなどを使う場合はそれぞれdata/<namespace>/<使用する機能>を選択してください。
    (例:advancementを使う場合、data/<namespace>/advancements という項目にチェックを入れてください)
